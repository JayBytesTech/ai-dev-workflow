# Contributing

Thanks for your interest in contributing to `ai-dev-workflow`.

## Workflow

1. Fork the repo and create a feature branch.
2. Keep changes focused and well-scoped.
3. Run `cargo fmt` and `cargo check` before opening a PR.
4. Include tests or update documentation where applicable.

## Style

- Prefer explicit behavior over hidden magic.
- Keep filesystem mutation minimal and well-scoped.
- Avoid changing user notes without clear, additive intent.

## Reporting Issues

Please include:

- OS and terminal environment
- version or commit hash
- the command you ran
- expected vs actual behavior
- any logs or transcripts you can share
